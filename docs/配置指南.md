# 📖 博客配置文件完全指南

> 本文档详细说明如何配置你的二次元个人博客

---

## 📁 文件结构

```
📁 你的博客目录/
├── index.html        # 主页面（一般不需要修改）
├── config.js        # ⭐ 配置文件（重点修改这里！）
├── style.css        # 样式文件（一般不需要修改）
├── script.js        # 功能脚本（一般不需要修改）
├── docs/            # 📄 文档目录
│   ├── tutorial/    # 教程分类目录
│   ├── notes/       # 笔记分类目录
│   ├── project/     # 项目分类目录
│   ├── thoughts/    # 随想分类目录
│   └── *.md         # 你的 Markdown 文档
└── *.pdf           # PDF 文件也可以直接放这里
```

---

## 🔧 config.js 详细配置

### 1. 博客基本信息

```javascript
blog: {
    title: "博客标题",
    // 显示在页面顶部的标题
    // 示例: "Akihabara Dream"

    subtitle: "副标题",
    // 显示在标题下方的简短描述
    // 示例: "Welcome to my anime dimension ✨"

    author: "你的名字",
    // 作者名称，会在首页展示
    // 示例: "OrdinarySaber"

    description: "博客描述",
    // 对博客的简要介绍
    // 示例: "一个二次元爱好者的个人博客"

    avatar: "头像图片地址"
    // 头像图片，支持以下格式：
    // 1. URL: "https://example.com/avatar.png"
    // 2. Base64: "data:image/png;base64,..."
    // 3. 本地路径: "images/avatar.png"
}
```

### 2. 社交链接配置（重点！）

```javascript
social: {
    // 只需要填写你有的链接，留空表示不显示

    github: "https://github.com/你的用户名",
    // GitHub 主页链接
    // 示例: "https://github.com/OrdinarySaber"

    bilibili: "https://space.bilibili.com/你的UID",
    // B站个人空间链接
    // 示例: "https://space.bilibili.com/123456789"

    email: "mailto:your@email.com",
    // 邮箱地址（mailto: 开头）
    // 示例: "mailto:example@gmail.com"

    twitter: "https://twitter.com/你的用户名",
    // Twitter 链接

    instagram: "https://instagram.com/你的用户名",
    // Instagram 链接

    youtube: "https://www.youtube.com/@你的频道",
    // YouTube 频道链接

    telegram: "https://t.me/你的用户名",
    // Telegram 链接

    csdn: "https://blog.csdn.net/你的ID",
    // CSDN 博客链接

    juejin: "https://juejin.cn/user/你的ID",
    // 掘金主页链接

    zhihu: "https://www.zhihu.com/people/你的ID",
    // 知乎主页链接
}
```

**配置示例：**

```javascript
social: {
    github: "https://github.com/OrdinarySaber",
    bilibili: "https://space.bilibili.com/123456789",
    email: "mailto:ordSaber@gmail.com",
    youtube: "",
    twitter: "",
    instagram: "",
    telegram: "",
    csdn: "",
    juejin: "",
    zhihu: ""
}
```

### 3. 文档分类配置

```javascript
categories: [
    // 分类列表，可以添加、删除、修改
    {
        id: "唯一标识符",
        // 分类的唯一ID，不能重复
        // 英文小写字母、数字、下划线
        // 示例: "tutorial"

        name: "显示名称",
        // 页面显示的分类名称
        // 示例: "教程"

        icon: "emoji图标",
        // 分类显示的图标
        // 示例: "📚"

        color: "#颜色代码"
        // 分类的强调色
        // 示例: "#ff6b9d"
    }
]
```

**内置分类（可直接使用）：**

| ID | 名称 | 图标 | 颜色 |
|----|------|------|------|
| tutorial | 教程 | 📚 | #ff6b9d |
| notes | 笔记 | 📝 | #c44dff |
| project | 项目 | 💻 | #4dffff |
| thoughts | 随想 | 💭 | #ffcc00 |
| translate | 翻译 | 🌐 | #4facfe |
| uncategorized | 未分类 | 📁 | #999 |

**添加自定义分类：**

```javascript
categories: [
    // ... 其他分类 ...

    {
        id: "anime",
        name: "动漫",
        icon: "🎬",
        color: "#ff6b9d"
    },
    {
        id: "music",
        name: "音乐",
        icon: "🎵",
        color: "#00d2ff"
    }
]
```

### 4. 文档配置（最重要！）

```javascript
documents: [
    {
        id: 1,
        // 文档唯一ID，数字类型，不能重复
        // 示例: 1

        title: "文档标题",
        // 文档显示的标题
        // 示例: "🎮 我的第一个游戏开发笔记"

        category: "分类ID",
        // 对应 categories 中的 id
        // 示例: "notes"

        date: "YYYY-MM-DD",
        // 发布日期
        // 示例: "2024-01-15"

        description: "简短描述",
        // 文档卡片的描述文字
        // 示例: "关于使用Unity开发2D游戏的入门笔记"

        tags: ["标签1", "标签2"],
        // 文档的标签数组
        // 示例: ["Unity", "GameDev", "入门"]

        file: "文件路径"
        // ⭐ 文档的实际文件路径
        // 相对于博客根目录
    }
]
```

## 📄 文档文件路径详解

### 方式一：放在 docs 目录（推荐）

```
📁 docs/
├── tutorial/
│   ├── getting-started.md
│   └── advanced.md
├── notes/
│   ├── unity-game-notes.md
│   └── ts-advanced.md
└── project/
    └── resume.pdf
```

**config.js 中配置：**

```javascript
// Markdown 文件
{
    title: "Unity 开发笔记",
    category: "notes",
    file: "docs/notes/unity-game-notes.md"
}

// PDF 文件
{
    title: "个人简历",
    category: "project",
    file: "docs/project/resume.pdf"
}
```

### 方式二：直接放在根目录

```
📁 博客根目录/
├── index.html
├── config.js
├── resume.pdf
├── tutorial.md
└── notes.md
```

**config.js 中配置：**

```javascript
// 根目录的 Markdown
{
    title: "使用教程",
    category: "tutorial",
    file: "tutorial.md"
}

// 根目录的 PDF
{
    title: "个人简历",
    category: "project",
    file: "resume.pdf"
}
```

### 方式三：使用外部URL

```javascript
{
    title: "GitHub 项目",
    category: "project",
    file: "https://example.com/docs/project.md"
}
```

## 📝 添加新文档步骤

### 步骤 1：创建文档文件

在 `docs/` 目录下创建 `.md` 文件：

```markdown
# 文档标题

## 第一章

内容...

## 第二章

内容...
```

### 步骤 2：在 config.js 中注册

打开 `config.js`，在 `documents` 数组中添加：

```javascript
documents: [
    // ... 现有文档 ...

    {
        id: 5,  // 新的唯一ID
        title: "🆕 新文档标题",
        category: "notes",  // 分类ID
        date: "2024-01-20",
        description: "简要描述",
        tags: ["标签1", "标签2"],
        file: "docs/新文档.md"  // 文件路径
    }
]
```

### 步骤 3：刷新页面

保存 config.js 后，刷新浏览器即可看到新文档。

## 🔍 常见问题

### Q1: 文档加载失败怎么办？

1. 检查文件路径是否正确
2. 确认文件确实存在于该路径
3. 如果是本地文件，确保在 docs 目录内
4. 检查浏览器控制台（F12）是否有错误信息

### Q2: 如何显示 PDF 文档？

PDF 文件配置和 Markdown 一样：

```javascript
{
    title: "个人简历",
    category: "project",
    file: "docs/resume.pdf"
}
```

点击卡片后会直接在新标签页打开 PDF。

### Q3: 如何删除某个社交图标？

将该项留空即可：

```javascript
social: {
    github: "https://github.com/xxx",
    bilibili: "",  // 这样就不会显示
    email: ""
}
```

### Q4: 如何修改博客主题颜色？

在 `theme` 对象中修改：

```javascript
theme: {
    primaryColor: "#ff6b9d",      // 主色调
    secondaryColor: "#c44dff",    // 副色调
    accentColor: "#4dffff",      // 强调色
    backgroundColor: "#0a0a12",   // 背景色
    cardColor: "#16161f",         // 卡片背景
    textColor: "#e0e0e0",        // 文字颜色
    mutedTextColor: "#888"       // 次要文字颜色
}
```

### Q5: 图片不显示怎么办？

1. **本地图片**：放在 `docs/` 或 `images/` 目录
2. **路径问题**：使用相对路径，如 `docs/images/avatar.png`
3. **URL图片**：使用完整URL
