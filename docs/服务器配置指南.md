# 🖥️ 服务器配置指南

> 本指南详细介绍如何本地测试和部署动态博客（支持在线编辑和评论功能）

---

## 架构说明

本项目采用**前后端分离**架构：

```
┌─────────────────┐         ┌─────────────────┐
│   前端 (静态)   │  ──▶   │   后端 (API)    │
│ localhost:8080 │         │ localhost:5096 │
└─────────────────┘         └─────────────────┘
        │                           │
        │   fetch('/api/...')       │
        └───────────────────────────┘
```

| 服务 | 技术 | 端口 | 说明 |
|------|------|------|------|
| 前端 | HTML + JS + CSS | 8080 | 用户访问的网站 |
| 后端 | ASP.NET Core | 5096 | 提供 API 接口 |

## 🚀 本地测试（动态模式 - 前后端分离）

### 准备工作

确保已安装：
- **Python** (用于前端静态服务器)
- **.NET SDK** (6.0 或更高版本，用于后端)

```powershell
# 检查版本
python --version
dotnet --version
```

### 启动步骤

#### 步骤 1：启动后端服务器（终端 1）

```powershell
# 进入后端项目目录
cd h:\TARE_Project\WEBTEST\BlogApi

# 运行后端
dotnet run
```

> 后端会在 http://localhost:5096 运行

#### 步骤 2：启动前端服务器（终端 2）

```powershell
# 进入项目根目录
cd h:\TARE_Project\WEBTEST

# 启动 Python 服务器（前端）
python -m http.server 8080
```

#### 步骤 3：访问网站

浏览器打开: **http://localhost:8080

### 配置说明

前端默认连接: http://localhost:5096/api

如需修改，在 script.js 开头修改 API_BASE 变量：
```javascript
const API_BASE = 'http://localhost:5096/api';
```

### 测试步骤

1. 启动后端 (终端 1): `dotnet run` → 运行在 http://localhost:5096
2. 启动前端 (终端 2): `python -m http.server 8080` → 运行在 http://localhost:8080
3. 浏览器打开 http://localhost:8080
4. 右上角应显示"服务器已连接"
5. 点击任意文档测试加载和编辑功能
6. 登录后可使用管理功能

## ☁️ 部署方案

### 方案一：部署到云服务器（推荐）

#### 1. 购买云服务器

- **阿里云/腾讯云**：轻量应用服务器（约 100 元/年）
- **配置建议**：1核2G内存

#### 2. 在服务器上部署

```bash
# 1. 安装 .NET SDK (Ubuntu/Debian)
wget https://dot.net/v1/dotnet-install.sh -O dotnet-install.sh
chmod +x dotnet-install.sh
./dotnet-install.sh --channel 8.0

# 2. 上传项目文件
# 使用 FTP 或 git clone

# 3. 配置 Nginx
sudo nano /etc/nginx/sites-available/default
```

**Nginx 配置示例：**
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        root /var/www/your-blog;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://localhost:5096/api/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection keep-alive;
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

#### 3. 启动后端服务

```bash
# 使用 Systemd 管理 .NET 应用
sudo nano /etc/systemd/system/blogapi.service
```

**Systemd 配置：**
```ini
[Unit]
Description=Blog API

[Service]
WorkingDirectory=/var/www/your-blog/BlogApi
ExecStart=/usr/bin/dotnet /var/www/your-blog/BlogApi/BlogApi.dll
Restart=always
User=www-data

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl enable blogapi
sudo systemctl start blogapi
```

### 方案二：使用 Docker 部署

#### 1. 创建 Dockerfile

在项目根目录创建 Dockerfile：
```dockerfile
FROM mcr.microsoft.com/dotnet/sdk:8.0 AS build
WORKDIR /src
COPY BlogApi/*.csproj BlogApi/
RUN dotnet restore BlogApi/BlogApi.csproj
COPY BlogApi/ BlogApi/
RUN dotnet publish BlogApi/BlogApi.csproj -c Release -o /app/publish

FROM mcr.microsoft.com/dotnet/aspnet:8.0
WORKDIR /app
COPY --from=build /app/publish .
COPY docs docs/
COPY config.js .
COPY index.html .
COPY style.css .
COPY script.js .
EXPOSE 5000
ENV ASPNETCORE_URLS=http://+:5000
ENTRYPOINT ["dotnet", "BlogApi.dll"]
```

#### 2. 构建和运行

```bash
# 构建镜像
docker build -t my-blog .

# 运行容器
docker run -d -p 80:5000 -v ./docs:/app/docs my-blog
```

访问 http://localhost 即可。

## 🔧 常见部署问题

### 1. 端口被占用

```powershell
# Windows - 查看端口占用 (后端 5096)
netstat -ano | findstr "5096"

# Windows - 查看端口占用 (前端 8080)
netstat -ano | findstr "8080"

# 结束进程
taskkill /PID <进程ID> /F
```

### 2. 防火墙问题

```powershell
# Windows 防火墙允许
netsh advfirewall firewall add rule name="BlogAPI" dir=in action=allow protocol=tcp localport=5096

# Linux 防火墙
sudo ufw allow 5096
sudo ufw allow 8080
```

### 3. 静态文件访问问题

确保以下文件与 BlogApi.dll 在同一目录：
- docs/ 目录
- config.js
- index.html
- style.css
- script.js

## ⚡ 一键启动脚本（推荐）

为了方便启动前后端服务，可以创建启动脚本：

### Windows (PowerShell)

创建 `start.ps1` 文件：

```powershell
# 启动前端
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd h:\TARE_Project\WEBTEST; python -m http.server 8080" -WindowStyle Normal

# 启动后端
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd h:\TARE_Project\WEBTEST\BlogApi; dotnet run" -WindowStyle Normal

Write-Host "前端: http://localhost:8080" -ForegroundColor Green
Write-Host "后端: http://localhost:5096" -ForegroundColor Green
```

### Windows (CMD)

创建 `start.bat` 文件：

```bat
@echo off
echo 启动前端服务...
start "Frontend" cmd /k "cd /d h:\TARE_Project\WEBTEST && python -m http.server 8080"

echo 启动后端服务...
start "Backend" cmd /k "cd /d h:\TARE_Project\WEBTEST\BlogApi && dotnet run"

echo.
echo 请访问: http://localhost:8080
echo.
```

### Linux/Mac

创建 `start.sh` 文件：

```bash
#!/bin/bash

# 启动前端
python -m http.server 8080 &

# 启动后端
cd BlogApi && dotnet run &

echo "前端: http://localhost:8080"
echo "后端: http://localhost:5096"
```

运行脚本：
```bash
chmod +x start.sh
./start.sh
```

## API 接口说明

部署后，以下 API 可用：

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/documents | 获取所有文档列表 |
| GET | /api/documents/{id} | 获取指定文档内容 |
| PUT | /api/documents/{id} | 更新文档内容 |
| GET | /api/comments/{docId} | 获取文档评论 |
| POST | /api/comments | 添加评论 |
| DELETE | /api/comments/{id} | 删除评论 |
